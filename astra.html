Project Overview

Solving real-world attendance challenges with smart technology.

The Problem

Traditional attendance methods are slow and vulnerable to "proxy attendance," wasting valuable class time and undermining academic integrity.

Our Solution

ARTAMS uses dynamic QR codes and GPS verification, accessible from a web interface with a real-time dashboard and data export capabilities.

Milestone 1: Core Logic & CLI Demo

Building and perfecting the backend logic in a simulated CLI environment.

1. The Two-Terminal Setup

Open two terminals: one for the **Teacher** and one for the **Student**. Run `./artams_cli` in each to simulate the real-time interaction.

2. Teacher Generates a Dynamic Token

In the **Teacher's terminal**, start a session. The CLI displays a secure, expiring token, simulating the QR code.

3. Student Enters Token & Location

In the **Student's terminal**, select "Mark Attendance" and enter Student ID, the token, and mock GPS coordinates.

4. Core Logic Validates Everything

The CLI validates the token, checks for duplicates, and verifies the coordinates. The result is printed in the student's terminal.

5. Attendance is Logged & Viewable

On success, the teacher can select "View Log" in their terminal to see a real-time list of all marked students.

Web Application Workflow

1. Teacher Starts Session: Clicks "Start Session" on the web dashboard to call the `/startSession` API.
2. Student Scans QR: The frontend displays a refreshing QR code. Student scans it, sending token and GPS data to the `/markAttendance` API.
3. Backend Validates: `server.c` uses core C functions to validate the token, location, and check for duplicates.
4. Live Dashboard Updates: On success, data is saved to Firebase, and the teacher's dashboard updates instantly.
5. Export Data: Teacher clicks "Export to Excel" to call a `/getReport` API and download a formatted sheet.

File Responsibilities

`core/student_db.c`: Manages student data using a Hash Table. Functions: `addStudent()`, `findStudent()`.

`core/token_manager.c`: Creates and validates session tokens. Functions: `generateToken()`, `validateToken()`.

`core/location_validator.c`: Validates GPS coordinates. Function: `isWithinRange()`.

`fastcgi/server.c`: Main backend entry point. Listens for HTTP requests and calls core functions.
