ARTAMS — Attendance Management System

A practical guide to building a reliable attendance CLI and web system for academic use.

Project Overview

Solving real-world attendance challenges with smart technology.

The Problem

Traditional attendance methods are slow and vulnerable to "proxy attendance," wasting valuable class time and undermining academic integrity.

Our Solution

ARTAMS uses dynamic QR codes and GPS verification, accessible from a web interface with a real-time dashboard and data export capabilities.

Project Milestones & Roadmap

A clear, phased approach to development. First, we perfect the core logic, then we build the web interface.

Milestone 1: File-Based CLI Application

Our first goal is to build a robust, standalone Command-Line Interface application. This version uses simple text files (`.txt`) as its database, allowing us to prove all core logic works flawlessly before moving to more complex systems.

CLI Demo Workflow

1. Build with Make

Run `make` in the terminal to compile all `.c` files into the `artams_cli` executable.

2. Teacher Starts Session

Teacher runs `./artams_cli`, chooses Teacher Mode, and starts a session. A token is generated and saved to `data/sessions.txt`.

3. Student Marks Attendance

In a separate terminal, the student runs `./artams_cli`, enters their roll number, the token, and mock coordinates.

4. Validation and Logging

The system validates all data against the `.txt` files and, if successful, appends a new entry to `data/attendance_log.txt`.

Milestone 2: Web App & Deployment

In the second phase, we will replace the file-based system with a FastCGI server and Firebase database, and build the final web interface.
